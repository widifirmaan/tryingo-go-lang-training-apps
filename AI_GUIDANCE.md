# Tryngo - AI Development Guidance

## Project Overview
Tryngo is an interactive coding education platform built as a React + Vite SPA with Tailwind CSS. It provides structured learning paths from absolute beginner to professional for 27 programming languages and frameworks.

## Tech Stack
- **Frontend**: React 19, Vite 6, Tailwind CSS 4
- **Animation**: Motion (motion/react)
- **Markdown**: react-markdown + remark-gfm
- **Icons**: lucide-react
- **Code Editor**: @monaco-editor/react
- **Search**: fuse.js with build-time index
- **Code Execution**: Cloudflare Sandbox SDK
- **Deployment**: Cloudflare Pages (wrangler.toml configured)

## Architecture

### Directory Structure
```
apps/web/
├── public/
│   ├── data/course/{slug}/{level}/{lang}/week{N}-{slug}.md  # 864 files
│   └── search-index.json              # Generated at build time
├── scripts/
│   ├── generate-course-md.mjs         # Curriculum generator
│   ├── generate-full-curriculum.mjs   # Full 864-file generator
│   └── build-search-index.mjs         # Search index builder
├── src/
│   ├── App.tsx                        # Main app with routing, modals, state
│   ├── assets/                        # Logos (SVG), images
│   ├── components/
│   │   ├── CoursePage.tsx             # Markdown course viewer + nav
│   │   ├── CodePlayground.tsx         # Interactive Monaco editor
│   │   ├── HeroSection.tsx            # Hero + sidebar nav
│   │   ├── TrackCard.tsx              # Course card (explore mode)
│   │   ├── ProductCardMale.tsx        # Featured card (hero mode)
│   │   ├── ProductCardFemale.tsx      # Featured card (hero mode)
│   │   └── Modals.tsx                 # Cart, Search, Detail, Settings
│   ├── data/
│   │   ├── tracksData.ts              # 27 track definitions
│   │   ├── materials.ts               # Week 1 material data (legacy)
│   │   └── curriculum.ts              # Curriculum metadata
│   └── utils/
│       ├── translations.ts            # id/en translations
│       └── search.ts                  # Fuse.js search utility

workers/
└── code-execution/                    # Sandbox SDK Worker project
    ├── src/worker.ts                  # POST handler for code execution
    ├── Dockerfile                     # Container with Go, Python, Node.js
    ├── wrangler.jsonc                 # Worker config with containers
    ├── package.json
    └── tsconfig.json
```

### State Management
- Simple React useState in App.tsx (no Redux/Zustand needed)
- Key state: `isExploring`, `activeCourseId`, `playgroundCode`, `playgroundLanguage`

### View Modes
1. **Hero Mode** (default): Large hero + 2 featured product cards
2. **Explore Mode** (GO clicked): Sidebar nav + horizontal/vertical card grid
3. **Course Mode** (Mulai Course): Sidebar + course content viewer + playground

## Curriculum Structure
Each language track has 4 levels × 4 weeks = 16 weeks total:
```
beginer/     → Beginner (weeks 1-4)   - No coding experience needed
intermediate/ → Intermediate (weeks 5-8)
advanced/    → Advanced (weeks 9-12)
pro/         → Professional (weeks 13-16)
```

Files are generated by `scripts/generate-full-curriculum.mjs` and stored in:
`public/data/course/{slug}/{level}/{lang}/week{N}-{topic}.md`

## Key Implementation Details

### CoursePage
- Reads markdown from `public/data/course/` via fetch
- Renders with react-markdown + custom `.lesson-body` CSS class
- Level/week selector in header
- "Try it yourself" button opens CodePlayground
- Styling matches reference at tryngo-web.pages.dev

### CodePlayground
- Monaco Editor with syntax highlighting (go, python, html, typescript)
- **Web languages** (HTML/CSS/JS/TS): Rendered via sandboxed iframe, console.log captured
- **Server languages** (Go, Python): POST to Cloudflare Worker (Sandbox SDK), output displayed
- Ctrl+Enter shortcut to run
- Fullscreen mode, code reset, dark VS Code theme
- Worker URL configured via `VITE_EXECUTION_WORKER_URL` env var

### Code Execution Worker (`workers/code-execution/`)
- Uses `@cloudflare/sandbox` SDK for secure, isolated code execution
- Docker container with Go, Python 3.11, Node.js 20
- Accepts POST `{ code, language }` → writes file → executes → returns `{ stdout, stderr, exitCode, success, duration }`
- Supports: python, go, javascript, typescript (via tsx)
- Deploy: `npx wrangler deploy` in workers/code-execution/
- Requires Docker locally and Cloudflare Workers paid plan (Containers)

### Search
- Build-time index: `scripts/build-search-index.mjs` extracts title, excerpt, body from all 864 .md files
- Index stored at `public/search-index.json` (~a few MB)
- Frontend `search.ts` loads index lazily via fetch + Fuse.js
- Two search tabs in SearchModal:
  - **Modul**: Track metadata filtering (category, sort, text)
  - **Materi Kursus**: Full-text fuzzy search across all materials
- Clicking a course content result navigates directly to that course

### Accessibility
- `lang` attribute on `<html>` syncs with language setting
- `role="application"` on root, `role="main"` on content area
- `aria-label` on navigation, main content, and interactive regions
- `focus-visible` keyboard outlines (green)
- Skip-to-content link (CSS only, toggled on focus)
- Meta description + theme color in `<head>`
- Preferences-based reduced motion via Motion library
- Color contrast meets WCAG AA for both light/dark modes

### Responsive Breakpoints
- Mobile: < 768px (single column, bottom navigation)
- Tablet: 768-1024px (2 columns)
- Desktop: > 1024px (sidebar + main content)

## How to Continue

### Generating Curriculum
```bash
cd apps/web
node scripts/generate-full-curriculum.mjs
```

### Rebuilding Search Index
```bash
cd apps/web
node scripts/build-search-index.mjs
```

### Deploying Code Execution Worker
```bash
cd workers/code-execution
npm install
npx wrangler deploy
```
Then set `VITE_EXECUTION_WORKER_URL` in apps/web/.env

### Adding a New Language Track
1. Add logo SVG to `src/assets/logos/`
2. Add entry to `src/data/tracksData.ts`
3. Add to `SLUG_MAP` in `CoursePage.tsx`
4. Add to generator script's `TRACKS` array
5. Regenerate curriculum

### Run Dev Server
```bash
cd apps/web
npm run dev
```

### Cloudflare Pages Deployment
```bash
cd apps/web
npm run build
npx wrangler pages deploy dist
```

## Design Principles
1. **Progressive disclosure**: Show complexity gradually
2. **Learn by doing**: Every concept has an interactive example
3. **Real-world focus**: Projects mirror actual industry needs
4. **Zero to pro**: Start from "what is a computer" to production deployment
