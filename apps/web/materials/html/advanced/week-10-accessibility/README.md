# Week 10: Web Accessibility (a11y)

## Overview
Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with the web. This week covers ARIA, semantic HTML, keyboard navigation, and screen reader compatibility.

## Learning Objectives
- Understand WCAG 2.1 guidelines and levels (A, AA, AAA)
- Apply ARIA attributes to enhance accessibility
- Write semantic HTML that works for all users
- Implement keyboard navigation patterns
- Test with screen readers

## Weekly Project
**Accessible Blog** — Convert a blog template into a fully accessible experience with proper ARIA landmarks, keyboard navigation, skip links, and screen reader announcements.

## Topics
1. ARIA Roles, States, and Properties
2. Semantic HTML Structure
3. Keyboard Accessibility
4. Screen Reader Testing and Best Practices
