# Week 22: Testing HTML

## Overview
Testing ensures HTML quality, accessibility, and cross-browser compatibility. This week covers validators, Lighthouse auditing, accessibility tools, and cross-browser testing.

## Learning Objectives
- Validate HTML with W3C Validator
- Run and interpret Lighthouse audits
- Use accessibility testing tools (axe, WAVE)
- Test across browsers and devices

## Weekly Project
**HTML Quality Dashboard** — Create a testing dashboard that validates HTML, runs accessibility checks, and reports issues.

## Topics
1. HTML Validation
2. Lighthouse Audits
3. Accessibility Testing Tools
4. Cross-Browser Testing
